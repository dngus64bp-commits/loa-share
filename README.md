# LOA-SHARE

로스트아크 레이드 데미지를 공대원들과 실시간으로 공유하고, 결과를 디스코드 채널에 박제하는 웹앱.

## 무엇이 들어있나

- **server.js** — Node.js + Socket.IO 서버. 같은 공대 ID로 접속한 사람들끼리 데미지 데이터를 중계.
- **public/index.html** — 사용자가 보는 웹페이지.
- **public/app.js** — 화면 캡처, 영역 선택, OCR, 서버 통신 로직.
- **package.json** — 의존성 정의.

## 핵심 작동 원리

1. 사용자가 브라우저에서 로스트아크 창을 화면 공유로 선택
2. 마우스 드래그로 전투분석기의 "내 데미지" 숫자 영역을 지정
3. 1.5초마다 그 영역만 잘라서 **Tesseract.js**로 OCR
4. 읽어낸 숫자를 서버로 보내면, 같은 공대 ID인 사람들에게 브로드캐스트
5. (옵션) 디스코드 웹훅 URL로 현재 순위를 채널에 박제

서버는 데이터를 **저장하지 않습니다.** 메모리 안에서만 중계하고, 연결이 끊기면 사라져요.

## 로컬에서 실행하기

```bash
# 1. 의존성 설치 (최초 1회)
npm install

# 2. 서버 실행
npm start

# 3. 브라우저에서 http://localhost:3000 접속
```

다른 PC와 테스트하려면 같은 와이파이에서 `http://본인PC의IP:3000`으로 접속하면 돼요. (방화벽 허용 필요)

## 인터넷에 배포하기 (무료)

### Render.com (추천)

1. https://render.com 가입 (GitHub 계정 가능)
2. 이 폴더를 GitHub 저장소에 올린다
3. Render에서 "New → Web Service" → 저장소 연결
4. 설정:
   - Build Command: `npm install`
   - Start Command: `node server.js`
   - Plan: Free
5. 배포되면 `https://내봇이름.onrender.com` 주소를 받음

### Railway

1. https://railway.app 가입
2. "New Project → Deploy from GitHub repo"
3. 자동으로 Node.js 감지하고 배포

## 디스코드 웹훅 만드는 법

1. 디스코드 채널에서 → 톱니바퀴(설정) → 연동 → 웹훅
2. "새 웹훅" → 이름과 채널 지정 → "웹훅 URL 복사"
3. 그 URL을 LOA-SHARE 페이지의 "웹훅 URL" 칸에 붙여넣기

## OCR 정확도 팁

- 데미지 영역을 **숫자만** 좁게 선택하세요. 너무 넓으면 다른 글자가 섞여서 오인식돼요.
- 화면 해상도가 높을수록 정확도 ↑
- 전투분석기 글자가 흰색 + 어두운 배경이면 가장 잘 인식돼요

## 주의사항

- LOANA처럼 화면 OCR 방식이라 게임 파일이나 메모리에 손대지 않습니다. 안전한 방식이에요.
- 다만 화면을 가리거나 글자가 흔들리면 인식이 튈 수 있어요. 그래서 10배 이상 점프하는 값은 자동으로 무시합니다.
- 실시간이라 서버는 가벼우면 충분합니다. Render 무료 플랜으로도 동작해요.
